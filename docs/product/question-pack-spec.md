# 질문팩 제품 명세 v0.4

## 1. 목적

질문팩은 팩 주인의 자기인식과 방문자의 관점을 빠르게 비교하고, 방문자를 동일한 팩의 다음 주인으로 전환하는 핵심 콘텐츠 단위다.

## 2. 팩 구성

하나의 팩 템플릿은 다음을 가진다.

- 제목과 커버
- 추천 관계 태그
- 분위기 태그
- 민감도
- A/B 질문 카드 10장
- 대표 질문인 Signature 카드 1장
- 기본 공유 권장값: 공개 또는 1:1
- 제작자와 리믹스 출처

팩 템플릿 제작자는 해당 템플릿을 사용한 다른 사람의 셀프 응답과 방문자 응답을 볼 수 없다.

## 3. 카드 구성

카드 한 장은 다음 값을 가진다.

- 주인용 1인칭 질문
- 방문자용 이름 치환 질문
- 왼쪽 선택지
- 오른쪽 선택지
- 주제·관계·민감도 태그
- Signature 여부
- 활성 상태와 버전

예시:

```text
주인: 서운한 일이 생기면 나는?
방문자: 서운한 일이 생기면 이 사람은?

왼쪽: 바로 말한다
오른쪽: 혼자 삭인다
```

주인용 질문과 방문자용 질문은 카드에 함께 저장한다. P0 공식 팩은 운영자가 두 문구를 직접 작성·검수한다. P1 제작 화면은 방문자용 3인칭 기본 문구를 제공할 수 있지만 AI로 질문을 새로 쓰지 않으며, 발행 전에 두 문구를 모두 확인·수정할 수 있어야 한다.

## 4. 카드 작성 규칙

- 관찰하거나 추측할 수 있는 행동과 태도를 묻는다.
- 한 카드에는 하나의 판단만 넣는다.
- 두 선택지는 겹치지 않고 동시에 참이 되기 어렵게 만든다.
- 한쪽이 명백히 더 도덕적이거나 매력적으로 보이지 않게 균형을 맞춘다.
- 질문은 모바일 카드에서 두세 줄 이내, 선택지는 각 두 줄 이내를 목표로 한다.
- 정답·친밀도·우열을 전제로 하지 않는다.
- 특정 개인의 비밀, 연락처, 성적 정보, 의료 정보 등 불필요한 개인정보를 요구하지 않는다.
- 썸·연애처럼 민감한 팩은 1:1 공유와 비공개 결과를 기본값으로 둔다.

## 5. 주인 응답

1. 주인이 팩을 선택한다.
2. 중간 대기 화면 없이 첫 질문부터 카드 10장에 답한다.
3. 모든 답변을 완료해야 링크를 만들 수 있다.
4. 공개 링크 또는 1:1 링크를 선택한다.
5. 링크에는 정답이나 셀프 응답 값을 포함하지 않는다.

## 6. 방문자 카드 배정

방문자는 필수 3장을 받는다.

1. 모든 방문자에게 동일한 Signature 카드 1장
2. Signature를 제외한 카드 중 해당 팩 플레이에서 응답 수가 가장 적은 그룹
3. 그 그룹에서 중복 없이 무작위로 고른 카드 2장

동률 무작위 선택은 방문자마다 달라질 수 있다. 이미 해당 방문자가 답한 카드는 다시 배정하지 않는다.

3장 제출 후 결과를 즉시 보여준다. 결과 화면에서 2장 추가 응답을 선택하면 아직 답하지 않은 카드 중 표본 수가 적은 카드부터 같은 방식으로 배정한다.

## 7. 관계 입력

방문자는 응답 전에 관계와 시점을 직접 선택한다.

관계 code와 한글 label은 다음 exact registry를 사용한다. DB와 analytics에는 code만 저장하고 label은 검수된 registry에서 derive한다.

| code | 한글 label |
|---|---|
| `old_friend` | 오래된 친구 |
| `school_friend` | 학교 친구 |
| `coworker` | 직장 동료 |
| `romantic` | 썸·연인 |
| `family` | 가족 |
| `online_friend` | 온라인 친구 |
| `social_follower` | SNS 팔로워·온라인에서만 봄 |
| `other` | 기타 |

알게 된 시점은 현재 시각 기준 서로 알게 되거나 팔로우하기 시작한 기간을 고른다. 구간은 비중첩이며 저장한 code는 선택 당시 구간을 뜻해 시간이 지나도 자동 변경하지 않는다.

| code | 한글 label | 경계 |
|---|---|---|
| `under_one_year` | 1년 미만이에요 | `< 1년` |
| `one_to_three_years` | 1년 이상 · 3년 미만 | `>= 1년, < 3년` |
| `three_to_five_years` | 3년 이상 · 5년 미만 | `>= 3년, < 5년` |
| `five_to_ten_years` | 5년 이상 · 10년 미만 | `>= 5년, < 10년` |
| `ten_years_or_more` | 10년 이상이에요 | `>= 10년` |
| `not_sure` | 잘 모르겠어요 | 구간을 고르기 어려움 |

공개 링크에서는 이름을 요구하지 않는다. 같은 관계 응답자가 3명 이상 모이기 전에는 공개 관계별 집계를 만들지 않는다.

방문자가 필수 응답을 제출하면 추측하기 어려운 비밀 응답 관리 링크를 발급해 현재 브라우저에 저장하고 복사할 수 있게 한다. 방문자는 계정이나 이름 없이 이 링크로 자신의 응답을 철회할 수 있다. 관리 링크를 잃으면 신원 확인이나 재발급을 제공하지 않는다. 철회하면 응답과 집계를 제거하고 링크를 폐기하며, 완료된 1:1 링크는 다시 열지 않는다.

## 8. 프로필 집계

- 유효하게 완료된 친구 응답 한 건을 프로필의 `시선` 한 건으로 센다.
- 친구 응답에 포함된 각 A/B 선택은 별도의 `카드 응답`으로 저장한다.
- 카드 응답은 팩 플레이·카드 버전·방문자 관계별로 왼쪽과 오른쪽 선택 수를 집계한다.
- 같은 관계의 유효한 완료 방문자가 3명 이상일 때만 관계 레이어를 공개 가능 상태로 만든다.
- 공개 가능한 관계에서도 같은 질문의 카드 응답이 3개 이상일 때만 좌우 선택 수를 표시한다.
- 두 기준 중 하나라도 부족하면 선택 수 대신 `시선을 모으는 중 · n/3`을 표시한다.
- 삭제·철회·무효 응답은 전체 시선 수와 모든 관계·질문 집계에서 제거한다.
- 최근 도착 정보는 방문자 신원이나 개별 선택이 아니라 `새 시선 도착`, `새 관계 도착`, `집계 공개 가능` 같은 익명 상태 변화로만 표시한다.
- 프로필은 원본 질문, 좌우 선택지, 셀프 선택, 관계별 선택 수를 사용하며 AI 요약·성격 문장·고정 유형을 만들지 않는다.
- 1:1 응답과 민감 관계 결과는 명시적 공개 선택 없이는 공개 프로필 집계에 포함하지 않는다.

P0 최소 프로필은 주인용 누적 화면이다. 필수 3장을 제출하지 않은 방문자에게 셀프 선택과 관계별 선택 수를 노출하지 않는다. 제출한 방문자는 자신이 답한 카드에 한해서만 비교 결과를 본다.

## 9. 비교 결과

- 필수 3장 제출 전에는 주인의 셀프 응답을 노출하지 않는다.
- 제출 후 각 카드에 `주인의 실제 답`과 `내가 본 주인`을 나란히 표시한다.
- 자신이 답한 카드의 같은 항목과 다른 항목, 방문자가 선택한 관계와 알게 된 시점을 표시한다.
- 서로 다른 선택이 있으면 Signature 카드의 차이를 우선하고, 없으면 팩 순서상 첫 차이를 `가장 흥미로운 차이`로 표시한다.
- 세 카드가 모두 같으면 차이를 생성하지 않고 `세 항목을 모두 같게 봤어요`로 표시한다.
- 단일 친밀도 점수나 순위를 만들지 않는다.
- Primary CTA는 `나도 이 팩으로 시작하기`다.
- Primary CTA는 동일한 팩의 셀프 10장 응답으로 바로 연결한다.
- Secondary CTA는 `2장 더 답하기`다.

## 10. P1 팩 제작

기본 제작은 빈 폼 입력이 아니라 선택과 검수다.

1. 관계·주제·분위기·민감도를 선택한다.
2. 시스템이 검증된 질문 후보 15장을 제안한다.
3. 사용자가 10장을 고르고 Signature 카드 1장을 지정한다.
4. 순서와 좌우 선택지를 확인한다.
5. 기본 제목을 확인하고 커버 템플릿을 선택한다.
6. 비공개 또는 링크 공개로 발행한다.

카드별 직접 수정과 새 카드 작성은 고급 기능으로 제공하되 필수 단계로 만들지 않는다.

## 11. 검수 기준

- 카드 수가 정확히 10장이다.
- Signature 카드가 정확히 1장이다.
- 모든 카드에 주인용·방문자용 질문과 좌우 선택지가 있다.
- 주인용·방문자용 문장이 자연스럽다.
- 선택지가 중복되거나 한쪽으로 유도하지 않는다.
- 민감도와 공유 기본값이 일치한다.
- 방문자 3장 배정에서 중복 카드가 없다.
- 방문자 제출 전 셀프 응답이 노출되지 않는다.
- 관계 레이어와 질문별 선택 수가 각각 최소 표본 기준을 충족하기 전에는 공개되지 않는다.
- 삭제·철회 후 전체 시선 수와 관계·질문 집계가 다시 계산된다.
- 제출 직후 비밀 응답 관리 링크가 발급되며, 이 링크로 철회하면 응답과 집계가 제거되고 링크가 폐기된다.
- 철회하더라도 완료된 1:1 링크는 다시 열리지 않는다.
- 결과와 프로필에 AI가 생성한 요약이나 성격 문장이 없다.
- 비교 결과에서 동일 팩 셀프 시작으로 한 번에 이동한다.
- 비교 결과의 가장 흥미로운 차이는 AI 없이 Signature 우선·팩 순서 규칙으로 결정되고, 차이가 없으면 일치 상태를 표시한다.

- `slug`: `old-friend`
- `version`: `old-friend-v1`
- `relationship_tag`: `old_friend`
- `tone`: `warm_reminiscence`
- `sensitivity`: `low`
