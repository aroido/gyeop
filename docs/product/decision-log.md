# 제품 의사결정 기록

## 2026-07-15 — 모바일 웹 우선

- 결정: P0는 모바일 웹으로 만들고 방문자 응답은 앱 출시 후에도 웹에서 가능하게 유지한다.
- 이유: Instagram·KakaoTalk 링크에서 설치나 로그인 없이 바로 응답해야 바이럴 퍼널이 유지된다.
- 결과: 네이티브 앱은 프로필 주인의 알림·관리 재방문이 검증된 뒤 별도 검토한다.

## 2026-07-15 — 주인 10장, 방문자 3장

- 결정: 팩 주인은 10장 전부에 답하고 방문자는 공통 핵심 1장과 균형 추출 2장만 필수로 답한다.
- 이유: 주인은 자기 프로필을 만들 동기가 있지만 방문자에게 10장을 요구하면 링크 전환이 떨어진다.
- 결과: 3장 완료 후 즉시 비교하며, 2장 추가 응답은 선택 사항으로 둔다.

## 2026-07-15 — 관계는 방문자가 직접 선택

- 결정: 팩 주인이 공유 전에 관계를 지정하지 않는다.
- 이유: 공개 링크는 친구뿐 아니라 온라인 친구와 SNS 팔로워도 참여할 수 있다.
- 결과: 방문자가 관계와 알게 된 시점을 직접 선택하고 관계별 프로필 집계에 사용한다.

## 2026-07-15 — 방문자를 다음 주인으로 전환

- 결정: 비교 결과의 Primary CTA는 `나도 이 팩으로 시작하기`다.
- 이유: 방문자는 단순 응답자가 아니라 다음 링크를 만드는 잠재적 프로필 주인이다.
- 결과: CTA를 누르면 탐색 화면 없이 동일한 팩의 셀프 10장 응답으로 이동한다.

## 2026-07-15 — 사용자 팩 제작은 P1

- 결정: P0는 공식 팩으로 루프를 검증하고 P1에 선택·조합형 팩 메이커를 추가한다.
- 이유: 질문 공급은 확장해야 하지만 첫 프로토타입에 공개 마켓까지 포함하면 핵심 가설 검증이 느려진다.
- 결과: P1 사용자는 관계·주제·분위기를 고르고 추천 15장 중 10장을 선택해 링크 공개 팩을 만든다.
